# Prompt: Gerar Resumo Executivo

> **Como usar:** Use para gerar um resumo do documento de requisitos para apresentar ao cliente,
> ao time de desenvolvimento ou a um novo membro que entrou no projeto.

---

## Variante A — Resumo Executivo para o Cliente

```
Com base em todos os documentos de requisitos do projeto FitnessApp, gere um resumo executivo para apresentar ao cliente.

O resumo deve:
- Ter no máximo 2 páginas (formato de leitura rápida)
- Usar linguagem não técnica
- Cobrir:
  1. Visão geral do app e seus objetivos
  2. Lista dos módulos e o que cada um faz
  3. Principais fluxos do usuário (em linguagem simples)
  4. Decisões que ainda precisam ser tomadas pelo cliente (tabela de pendências)
  5. Próximos passos

Formato: Markdown limpo, adequado para converter em PDF ou apresentação.
```

---

## Variante B — Briefing para Desenvolvedor

```
Com base nos documentos de requisitos do FitnessApp, gere um briefing técnico para um desenvolvedor que vai começar a trabalhar no projeto.

Inclua:
1. Arquitetura de telas (lista completa com rotas sugeridas)
2. Principais entidades de dados identificadas (User, Workout, Routine, Exercise, Assessment...)
3. Integrações e APIs mapeadas
4. Pontos de atenção técnica identificados nas notas dos documentos de tela
5. O que está confirmado vs. o que ainda é pendente
6. Status atual: quais telas estão documentadas e quais faltam

Seja técnico e direto — este é para um desenvolvedor, não para o cliente.
```

---

## Variante C — Status Report de Progresso

```
Leia os documentos de requisitos do FitnessApp e gere um status report do andamento da documentação.

Inclua:
- Total de telas: [X] documentadas / [Y] total
- Status por módulo (tabela)
- Lista de pendências críticas (⚠️ PENDENTE em campos importantes)
- Estimativa: com base no ritmo atual, quanto falta para concluir?
- Sugestão de próximos 3 arquivos a documentar (pelos mais críticos para o fluxo principal)
```
